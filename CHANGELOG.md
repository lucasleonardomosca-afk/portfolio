## Versioning Strategy

### Semantic Versioning
- **MAJOR**: Significant changes to portfolio content or design
- **MINOR**: New features or enhancements (components, pages, sections)
- **PATCH**: Bug fixes and minor improvements

### Release Process
1. All changes committed to feature branches
2. Pull request review required before merge to main
3. Commits must be GPG-signed
4. Tag releases with version number (vX.Y.Z)
5. Update CHANGELOG.md with release notes
6. Push tags to origin
